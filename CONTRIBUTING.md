# Contributing to App Blueprint Docs

Thank you for helping improve App Blueprint Docs! This guide explains how to contribute.

---

## 🐛 Found an Issue?

If you found a typo, outdated information, or a confusing section:

1. **Open an Issue** on GitHub describing the problem
2. **Be specific:** Quote the problematic section and explain what's wrong
3. **Suggest a fix:** If you have an idea, include it

Example:
```
Title: PRD template is missing user persona template

Issue: The "Target Users" section doesn't have a template for defining personas
Expected: Should show an example like the other sections
Suggested fix: Add this template... [provide example]
```

---

## ✨ Have an Improvement?

If you want to improve a template, add a new document type, or enhance the guides:

1. **Fork the repository**
2. **Create a branch** for your changes: `git checkout -b improve/better-prd`
3. **Make your changes**
4. **Test your changes** (does it make sense? is it clear?)
5. **Submit a pull request** with a clear description

---

## 📝 Document Contribution Guidelines

When creating or improving documents, follow these principles:

### 1. Keep It Concise
- **Good:** Clear, short explanations with examples
- **Bad:** Long-winded prose that could be skimmed in 5 minutes

### 2. Show Examples
- Every major section should have at least one example
- Examples should be realistic and useful
- Include what "good" looks like and what "bad" looks like

### 3. Use Consistent Formatting
- Use Markdown headers correctly (H1, H2, H3)
- Use code blocks for technical content
- Use tables for comparing options

### 4. Make It Actionable
- Documents should help people actually build something
- Include AI prompts that can be directly copy-pasted
- Provide templates people can fill in

### 5. Stay Technical But Accessible
- Avoid unnecessary jargon
- If you use technical terms, explain them
- Write for both technical and non-technical readers

## 🎨 Template Structure

All document templates should follow this structure:

```markdown
# [Document Name]

**[One-line description of what this document does]**

[Overview paragraph]

---

## Section 1

Explanation + Example + Your Input fields

---

## Section 2

Explanation + Example + Your Input fields

---

## 🤖 AI Prompt to Generate This Document

[Copy-paste-able prompt for AI tools]

---

## 📝 Notes

[Tips and best practices]

---

**[Final motivational line]**
```

---

## 🚀 Adding a New Document Type

If you want to create a new document template (beyond the existing 5):

1. **Discuss first:** Open an issue proposing the new document
2. **Explain the value:** Why should every app have this document?
3. **Show an example:** Create a draft template
4. **Get feedback:** Team will review and suggest improvements
5. **Add to README:** Update the main README with the new document

---

## 🔍 Code of Conduct

- Be respectful and constructive
- Assume good intent
- Provide helpful feedback
- No spam or self-promotion
- Keep discussions focused on improving the docs

---

## 📋 Checklist for Pull Requests

Before submitting a PR, make sure:

- [ ] Spelling and grammar are correct
- [ ] Examples are realistic and helpful
- [ ] Formatting is consistent with other documents
- [ ] AI prompts are copy-paste-able
- [ ] No external links are broken
- [ ] Changes don't duplicate existing content
- [ ] I've tested this locally (if applicable)
